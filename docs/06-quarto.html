Reproducible data analysis in RStudio

Learning Objectives

  • Use Quarto for reproducible reporting
  • Use git for version control

Create a Quarto document

Quarto documents use markdown syntax to format text using simple symbols. For example, you can create headings by using the # symbol, create lists by using - or *, and create links by using [text](url).

RStudio has built in support for two types of markdown documents, RMarkdown and Quarto. Quarto is a next-generation version of RMarkdown that is designed to be more flexible and powerful. Quarto documents can include code chunks from many different programming languages including R, Python, Julia, and JavaScript.

Quarto documents can be rendered to many different formats including HTML (default), PDF, and Word.

RStudio provides you with a quarto file with the same name as your project. You can open this or create a new Quarto document by going to File > New File > Quarto Document.

Open the document. You will see that the document has a header at the top that includes the title. The header can also be used to specify an author, subtitle, date, and an output format of your document. You can also add other options to the header such as a table of contents, theme, and more.

Save the file in your project directory.

Render the file

You can render the document as is by clicking the Render button in the toolbar. This will create an HTML file in your project directory with the same name as your Quarto document. Open the HTML file in your web browser to view the rendered document.

Edit the file

Let’s make some changes to the document. In the document toolbar, you will notice a Visual and Source button. The Visual editor allows you to edit the document in a WYSIWYG (What You See Is What You Get) interface, while the Source editor allows you to edit the raw markdown code. You can switch between the two editors by clicking the buttons in the toolbar.

We are going to edit in the Source editor, so click the Source button. This will allow us to get to grips with the basics of markdown syntax. The Visual editor is great for quickly editing documents and has some powerful shortcuts to build html content, but it is not as flexible as the Source editor. You can switch between the two editors at any time.

Markdown basics

Further learning

The Quarto guide is a comprehensive resource for formatting your Quarto documents.

Use the Markdown basics page as a reference to build content for your Quarto document.

Text formatting

Begin by adding some text to the top of your document. You can format text using the following symbols:

  • *italic* for italic text
  • **bold** for bold text
  • ***bold italic*** for bold italic text
  • ~~strikethrough~~ for strikethrough text
  • code for code style text
In this study, we investigate the effects of **MOV10** overexpression in *HEK293F* cells using `DESeq2`.

Headings

We can create headings by using the # symbol. The number of # symbols indicates the level of the heading. For example, # is a top-level heading, ## is a second-level heading, and so on.

Add a heading for the introduction section of your document.

## Introduction

Links

Web links can be added using the following syntax: [text](url). For example, to add a link to the Quarto website, you can use the following code:

The data from this analysis is part of a larger study described in [Kenny PJ et al., Cell Rep 2014](https://pubmed.ncbi.nlm.nih.gov/25464849/).

Images

You can place images in your document using the following syntax: ![alt text](image_url). For example, to add an image from a URL, you can use the following code:

![](https://bioconductor.posit.co/images/logo/svg/Logo.svg)

If the image is stored on your computer, you can save it in your project directory and use a relative path to the image file.

Create a folder called images in your project directory and save an image in that folder. You can use the following code to add the image to your document:

![]("images/my_image.png")

You can add a caption to the image by adding a title in the square brackets. You can format the image by adding style attributes in the curly braces. For example, to add a caption and set the width of the image to 50%, you can use the following code:

![bioconductor_logo](images/bioconductor.png){width="25%" fig-align="center"}

Lists

Lists can be created using - or * for unordered lists and numbers for ordered lists. For example, to create a list, you can use the following code:

The RNA-seq analysis was performed using the following steps:
  
  1.  Quality control with [fastqc](https://www.bioinformatics.babraham.ac.uk/projects/fastqc/)
  2.  Adapter trimming with [cutadapt](https://cutadapt.readthedocs.io/en/stable/)
  3.  Pseudo-alignment with [salmon](https://combine-lab.github.io/salmon/) to the human transcriptome (GRCh38 Ensembl release 106).

Tables

Tables can be created using the following syntax.


The DESeq2 dataset contains counts for the following samples:

| Dataset    | Description           | Replicate   |
|------------|-----------------------|-------------|
| Control_1  | Control               | replicate 1 |
| Control_2  | Control               | replicate 2 |

It is sometimes easier to use the visual editor to create tables. You can switch to the visual editor by clicking the Visual button in the toolbar. You can then click the Insert Table button in the toolbar to create a table.

If you want to display a large table, it will be better to use a code block, read the data into R and display it in the document. We will see examples of this below.

Render your document again to see the formatted text in the introduction.

Callout blocks

You can add callout blocks to your document to highlight important information. Callouts are boxed sections of text that can be used to draw attention to important information. You can create a callout block using the following syntax:


:::{.callout-note}
This data has already been loaded into a DESeq2 dataset using `tximport` and saved as an RDS file. You can load the dataset into R using the `readRDS()` function.
:::

Further learning

Quarto has several types of preset callout blocks or you can create your own.

Code blocks

The real power of Quarto is the ability to include code blocks in your document. Code blocks are sections of code that can be executed when the document is rendered.

First, add a new header to your quarto document for the data analysis section:

## Data analysis

Code blocks are created using three backticks followed by the language of the code block. It is also good practice to name the code block. For example, to create a code block for R, you can use the following syntax:

\`\`\`{r load_libraries}

# Load the libraries we will use in this analysis
library(tidyverse)
library(DESeq2)
library(plotly)
library(pheatmap)
library(RColorBrewer)
library(plotly)
library(apeglm)
library(DT)
library(ggrepel)

# Load the DESeq2 dataset
dds <- readRDS("data/MOV10_dds.rds")
# Display the column data of the DESeq dataset
colData(dds)

\`\`\`

Render the document again to see the code block and the output of the code. You will see a bunch of text output relating to the loading of our libraries. This is default R output but isn’t very informative for anyone reading the document.

Execution options

The execution of code blocks in Quarto can be controlled using options that are specified at the top of each block. For example

  • echo: false will hide the code in the output
  • output: false will hide the output of the code block
  • message: false will hide messages generated by the code block
  • warning: false will hide warnings generated by the code block
  • error: false will hide errors generated by the code block
  • include: false will hide both the code and the output of the code block
  • eval: false will prevent the code block from being executed
    • This is good if you want to show the code but not run it in the rendering

To turn off messages and warnings, we can add the following to our code block:

 
\`\`\`{r load_libraries}
#| warning = FALSE
#| message = FALSE
# Load the libraries we will use in this analysis
library(tidyverse)
library(DESeq2)
library(plotly)
library(pheatmap)
library(RColorBrewer)
library(plotly)
library(apeglm)
library(DT)
library(ggrepel)

# Load the DESeq2 dataset
dds <- readRDS("data/MOV10_dds.rds")
# Display the column data from our dataset
colData(dds)


\`\`\`

Render the file again. Play around with some of the other code block options to see how they affect the output of your document.

Quarto has many options for execution control.

Quarto header

Let’s add some more information to the header to control the output of our document. We can add a table of contents, set the theme, and add a date to the header. The header should look like this:

---
title: "RNA-seq analysis"
author: "Your Name"
date: "`r Sys.Date()`"
format:
  html:
    toc: true
    toc-depth: 2
    theme: cosmo
    code-fold: false
    output-file: index.html
---

We could have used plain text for our date, but by using 2026-06-18 we can automatically update the date to the current date when the document is rendered. This is the R function for the current date. You can use any R function in the header by using the r prefix.

The table of contents and theme are added as html options. The table of contents will show all of the headings in the document. The toc-depth option controls how many levels of headings are shown in the table of contents. In this case, we have set it to 2, which means that only headings with 1 or 2 # symbols will be shown in the table of contents.

The theme controls the appearance of the document. You can choose from several preset themes to change the look and feel of your document.

We are also going to set the output file name to index.html. This is the default file name for the home page of a website. If we want to publish a single document as a website, we need to set the output file name to index.html.

Render your document again, open index.html to see how your document has changed.

Try changing code-fold to true and see how this affects the output of your document.

Further learning

It is also possible to add custom CSS to your document to change the appearance of your document. CSS styling is the standard way to format web pages.

HTML widgets

Because Quarto documents are rendered as HTML, we can include interactive HTML widgets in our document. For example, we use the plotly package to create an interactive plot that can be zoomed and panned. We also use the DT package to create interactive tables.

The DT package creates data tables that can be searched, sorted and filtered. The datatable() function takes a data frame as input and returns an HTML widget that can be rendered in the Quarto document. You can specify many options for the datatable, including the number of rows to display, the column names, and apply formatting to the table.

Further learning

The DT package has many options for creating interactive tables with custom formatting and styling. The gt and reactable packages are also great for creating html tables in Quarto documents.

Version control with git

When we set up our project, we instructed RStudio to create a git repository for our project. Git is a version control system that allows us to track changes to our code and documents. It is a powerful tool for software development, managing projects and collaborating with others.

  • Keep track of changes to your code and documents
  • Revert to previous versions of your code and documents
  • Collaborate with others without overwriting each other’s work or working on different versions of the same file
  • Publish your work to GitHub for sharing with others

You can link a git repository to a remote online repository on websites like GitHub and GitLab. This allows you to share your work with others and collaborate on projects. You can also use GitHub to host your Quarto documents and make them available to the public.

Git is a command line tool, but RStudio has built in support for git. You can use the git tab in RStudio (top right) to view the status of your project, commit changes you have made to documents and push changes to a remote repository.

We could do a whole course on git and GitHub, but for this lesson we will just show you how to track changes in your project.

Git ignore

When we created our project, RStudio automatically created a .gitignore file for us. This file tells git which files and folders to ignore when tracking changes to our project. The .gitignore file is important because it prevents us from adding unnecessary files or really large files to our git repository.

Commit changes

Go to the Git tab in RStudio. You will see a list of files in your project that have been modified (M). You can select the files you want to commit and click the Commit button. Committing a file means that you are saving a snapshot of the file at that point in time. You can commit as many times as you want, and each commit will create a new snapshot of the file.

This will open a new window. Enter a commit message: “Initial Commit”. Click the Commit button to commit your changes.

Open your quarto document, make some changes to the text and save the file . You will see that the quarto document is now listed as modified (M). Select the file and click the Commit button. Enter a commit message that describes the changes made and click Commit. Useful commit messages e.g. “Added introduction section”, “Updated figure 1”, “Fixed bug in volcano plot” will help you keep track of the version history of your project.

You can view the commit history by clicking the History button in the git tab. If you need to revert a file to a previous version, you can do so by right clicking on the file in the history and selecting Revert from the More menu.

Push changes to GitHub

Once you have committed your changes, you can push them to a remote repository on GitHub. Linking a project to a remote repository can be a bit of an arduous process, so we will not cover it here. The steps are outlined below.

  1. Create a GitHub account
  2. Create a personal access token on GitHub to allow you to securely connect to your GitHub account from RStudio
  3. Create a new repository on GitHub with the same name
  4. Copy the code to link your local repository to the remote repository on GitHub and run this in the RStudio terminal
  5. Push your committed changes to GitHub from the Git pane

Further learning

These instructions and the Quarto guide to GitHub pages outline the process of connecting your RStudio git project to GitHub.

Quarto publishing

Quarto documents need to be hosted on a web server to be accessible to others. There are many options for hosting Quarto documents, including GitHub pages, Netlify, and RStudio Connect. For this lesson, we will use GitHub pages to host our Quarto document.

GitHub pages is a free service provided by GitHub that allows you to host static websites directly from your GitHub repository.

  1. Go to your GitHub repository and click on the Settings tab.
  2. Scroll down to the GitHub Pages section and select the branch you want to use for GitHub pages. You can choose the main branch or a different branch.
  3. Select the folder you want to use for GitHub pages. You can choose the root directory or a folder called docs, root is fine for this project.
  4. Click Save to save your changes.

Go to the actions tab in your GitHub repository. You will see that GitHub has automatically created a workflow to build your Quarto document. This workflow will run every time you push changes to your repository. You can view the status of the workflow and see the logs for each step. Once the build completes successfully, you can view your Quarto document on GitHub pages by going to: https://<your-github-username>.github.io/<your-repo-name>

The link is also available in the GitHub Pages section of the Settings tab in your GitHub repository.

Quarto configuration file

When we created our project, RStudio automatically created a _quarto.yml file for us. This file contains the configuration settings for our Quarto project. You can edit this file to change the settings for your project.

For example, you can change the title of your project, the theme and many other settings. Settings here will be applied to all Quarto documents in your project. You can override these settings in the header of individual Quarto documents.

Edit your _quarto.yml file to change the theme and remove the theme from the header of your index.qmd file. The _quarto.yml file should look like this:

project:
  title: "RNA_seq_analysis"
format:
  html:
    theme: 
      - minty
editor: visual

Render your quarto document again to see how the theme has changed. You can choose from many different themes to change the look and feel of your document. You can also create your own custom theme by creating a CSS file and adding it to your project.

It is possible to build a Quarto website by creating multiple Quarto documents in your project, linking them together and adding a navigation bar to the _quarto.yml file. If you create a new project as a Quarto website, RStudio will create a _quarto.yml file with a navigation bar and a folder structure for your website.
